# LinkPilot

一个轻量级、社区驱动的影视与泛娱乐资源导航站，旨在解决优质内容入口分散、失效链接难以追踪、新站点发现成本高的问题。项目以“人工精选 + 机器可用性探测”为核心，为内容消费者、剪辑素材采集者、影视爱好者提供高可用、低延迟的外链聚合服务。本项目本身不存储任何版权内容，仅收录公开可访问的第三方页面链接，并以技术手段辅助用户判断链路健康度。

## 功能概览

- 全自动链路存活检测：每日定时对收录的所有外链发起 HEAD/GET 轻量探测，自动标记超时、4xx、5xx 异常状态，并在前端进行视觉警示。
- 被动式用户反馈聚合：允许访客提交“链接失效”、“跳转异常”、“内容不符”三类反馈，后台汇总后辅助人工下架或修正条目。
- 多维标签过滤系统：每条外链可附带“蓝光”、“中字”、“极速”、“备用线路”等标签，支持多标签组合筛选，提升检索效率。
- 分类目录树导航：按“电影”、“电视剧”、“动漫”、“纪录片”、“综艺”等一级分类组织资源，每类下设“近期热门”、“高分片库”、“即将下架”等动态子集。
- 简洁的管理后台：提供基于 IP 白名单 + 简单口令的轻量管理界面，用于增删改链接、编辑备注、查看历史探测日志。
- 开放 JSON API：提供 `/api/v1/links` 端点，支持按分类、标签、存活状态查询数据，便于其他开发者二次集成或制作客户端。

## 应用场景

- 影视爱好者在寻找某部冷门老片的在线观看入口时，可通过分类树快速定位到多个候选站点，并参考存活状态标识优先选择可用的链路。
- 视频剪辑师需要下载无字幕预告片或高清片花作为素材，通过“蓝光”+“极速”标签筛选，能迅速获得一批高带宽资源的直链索引。
- 运维或测试人员可将 LinkPilot 作为外部依赖监控面板，利用其 JSON API 定时拉取指定链接状态，集成到自有告警系统中。
- 新手用户通过浏览“热门推荐”和“近期更新”子集，能快速了解当前互联网上质量较高的影视资源门户，降低个人搜索成本。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境，前提是已安装 Git、Node.js 18+ 与 pnpm。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkpilot.git
cd linkpilot

# 安装依赖（使用 pnpm 以获得更快的速度和更少的磁盘占用）
pnpm install

# 复制环境变量模板，并修改数据库连接等配置
cp .env.example .env

# 初始化 SQLite 数据库表结构（仅首次运行）
pnpm run db:init

# 启动开发服务器（默认监听 3000 端口）
pnpm run dev
```

生产环境部署建议使用 `pnpm run build` 构建静态文件，并通过 PM2 或 Docker 运行 `server.js`。

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或 20.x LTS | 是 | 运行时环境，推荐使用 Active LTS 版本 |
| pnpm 8.x 或 9.x | 是 | 包管理工具，用于依赖安装与脚本执行 |
| SQLite 3 | 是 | 内嵌数据库，无需额外安装，但需确保文件写入权限 |
| 可用磁盘空间 ≥ 500 MB | 是 | 用于存放数据库文件、日志及临时探测缓存 |
| 互联网出口 | 是 | 用于外链存活探测，需允许出站 HTTP/HTTPS 请求 |
| 可选：Redis 6.x | 否 | 若启用分布式探测队列或会话缓存，需配置 Redis 连接 |

## 贡献指南

1. 在 GitHub 上 fork 本仓库，并切换到 `develop` 分支创建您的特性分支 `feat/your-feature-name`。

## 常见问题

Q: 为什么有些链接在列表中显示为“不可达”，但我本机浏览器可以打开？
A: 探测服务运行于云服务器节点，可能受目标站点地域屏蔽、Cloudflare 防护或反爬策略影响。您可以点击“手动复核”按钮触发即时探测，或提交“误报”反馈供管理员调整探测参数。

Q: 项目是否会存储或缓存第三方站点的视频内容？
A: 不会。LinkPilot 仅记录 URL 元数据（标题、分类、标签）以及响应状态码、响应时间等链路质量指标，不代理、不缓存任何媒体流数据。所有内容访问均跳转至原始站点。
