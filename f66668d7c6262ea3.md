# LinkPilot

LinkPilot 是一个面向技术团队与内容创作者的开源外链资源聚合与导航系统。该项目旨在解决个人与团队在项目开发、内容研究及学习过程中，分散收藏、难以检索、链接失效等痛点，通过结构化的数据组织与简洁的展示层，将零散的 URL 转化为可复用、可共享、可维护的知识资产。LinkPilot 定位于轻量级部署，既可作为个人浏览器的起始页，也可作为团队内部的技术资源索引中心。

项目默认提供一套精选的流媒体与影视技术相关的外部资源导航，并允许开发者通过简单的 JSON 或 YAML 配置文件自由增删分类与链接，实现导航站点的快速重建与定制。LinkPilot 不依赖复杂的前端框架，核心渲染逻辑基于原生 JavaScript 与模块化 CSS，确保在低版本浏览器及老旧设备上仍能保持基础可用性，适合对页面响应速度与兼容性有要求的用户群体。

## 功能概览

- **多级分类导航**：支持无限层级的分类目录结构，用户可根据资源类型、使用频率或项目阶段自定义分组，便于大规模链接的条理化组织。

- **链接状态检测**：内置轻量级链接可用性检查模块，可定时或手动触发对已收录 URL 的 HEAD 请求，以标记失效或响应超时的链接，辅助管理员清理或更新资源。

- **全文检索与快速跳转**：提供基于标题、描述及标签的模糊搜索功能，并支持键盘快捷键唤起搜索框，实现从查找到访问的高效路径。

- **导入与导出机制**：支持将当前导航数据导出为 JSON 或 CSV 格式文件，便于备份、版本控制或迁移至其他实例；同时支持从外部 JSON 文件或 URL 批量导入链接数据。

- **视觉主题定制**：内置明暗两套配色方案，并开放 CSS 变量接口，允许开发者覆盖主色、字体、间距等样式属性，以匹配现有品牌或个人审美。

- **访问统计看板**：基于 localStorage 记录各链接的点击次数与最后访问时间，在管理界面提供简单的热度排序视图，帮助识别高频使用的核心资源。

## 应用场景

- **技术团队内部知识库入口**：研发团队可将项目文档、设计稿链接、API 参考手册、CI/CD 流水线地址等统一收录至 LinkPilot，作为团队浏览器的默认新标签页，降低新人上手与日常协作中的信息查找成本。

- **内容创作者选题素材库**：视频博主、自媒体运营者可将竞品主页、素材网站、热点榜单、数据分析平台等外链按选题方向分类，配合检索功能快速调取参考资料，提升内容产出效率。

- **个人学习路径管理**：自学者可将在线课程、技术博客、官方文档、社区讨论帖按照学习阶段（入门、进阶、实战）归档，并结合链接状态检测功能定期检查课程链接是否失效，避免学习计划中断。

- **活动与会议资源聚合**：线下技术沙龙或线上峰会的组织者可使用 LinkPilot 搭建临时导航页，整合演讲嘉宾资料、直播入口、问卷表单、会后回放地址，参会者通过单页即可获取所有必要信息。

## 快速开始

以下指令适用于 Linux/macOS 及 Windows WSL 环境，请确保已安装 Git 与 Node.js（v16 或以上版本）。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkpilot-org/linkpilot.git

# 进入项目根目录
cd linkpilot

# 安装依赖（使用 npm）
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行完上述步骤后，在浏览器中访问 `http://localhost:3000` 即可查看默认导航页面。如需构建生产环境静态文件，请使用 `npm run build` 命令，产物将输出至 `dist` 目录，可部署至任何静态托管服务。

## 常见问题

**Q：如何在不重新构建整个项目的情况下，快速更新导航中的链接列表？**

A：若无需修改分类结构或样式，可直接编辑 `src/config/defaults.json` 文件，按现有格式增删或修改链接对象。保存后刷新浏览器页面即可生效（开发模式下）。对于生产环境，建议通过管理界面提供的导入功能上传新的 JSON 文件，系统会自动覆盖当前数据。

**Q：链接状态检测模块显示部分站点为不可用，但我确认浏览器可以正常访问，是什么原因？**

A：检测模块默认使用 HEAD 请求并设置 5 秒超时限制。某些站点可能屏蔽 HEAD 请求或响应速度较慢，导致误报为失效。您可以在 `src/core/checker.js` 中将请求方法调整为 GET，或延长超时阈值。同时请确保部署环境允许出站网络请求，且未被防火墙或 CORS 策略拦截。

**Q：能否将 LinkPilot 部署到 GitHub Pages 或 Cloudflare Pages 这类静态托管服务？**

A：完全可以。执行 `npm run build` 后，将生成的 `dist` 目录下的所有文件上传至托管服务即可。需注意，由于纯静态部署没有后端服务，导入导出功能将仅依赖于浏览器的 localStorage，无法进行服务端持久化。如需多设备同步数据，可考虑配合第三方的 JSON 存储服务或自建后端 API。
