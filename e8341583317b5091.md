# HyperLink Navigator

HyperLink Navigator 是一个面向技术调研者、内容聚合运营者以及互联网资源管理人员的开源外链资源整理与导航工具。项目本身不存储、不转发任何第三方内容，仅以结构化方式收集并呈现互联网公开的多媒体资源入口，帮助用户快速定位特定类型的流媒体信息站点。

本项目的目标用户包括：需要进行竞品站点分析的运营人员、希望快速获取可用影音资源索引的普通用户、以及希望基于现有外链列表二次开发自有导航站点的开发者。HyperLink Navigator 通过固定格式的 Markdown 文档和 JSON 索引文件，将散落的网络资源链接整理为可读、可维护、可扩展的开放式数据集。

本项目不提供任何破解、盗版或非法内容，所有收录链接均为互联网公开可访问地址，项目本身仅作为网址导航用途，不承担任何第三方站点内容的法律责任。

## 功能概览

- 外链分类索引：按站点类型、内容主题、访问频率等维度对收录的 URL 进行标签化分类，支持多级筛选。

- 链接可用性检测：内置简单的 HTTP 状态码检测脚本，可定期检查收录链接是否有效，并标记异常条目。

- Markdown 原生渲染：所有资源列表以纯 Markdown 书写，兼容 GitHub、Gitee 及各类静态站点生成器，无需额外解析。

- JSON 数据导出：提供与 Markdown 列表同步的 JSON 格式数据文件，方便开发者用于二次开发或自动化处理。

- 自定义标签系统：每个链接可附加多个自定义标签，用户可根据自身需求重新组织分类逻辑。

- 版本化变更记录：每次链接增删改均通过 Git 提交记录体现，支持回溯任意历史版本的资源列表。

- 轻量级部署：项目本身为纯静态文档，无需数据库和后台服务，可托管于任意 Web 服务器或代码托管平台。

## 应用场景

场景一：技术调研人员快速获取竞品站点列表。在进行流媒体或内容分发领域的竞品分析时，调研人员可通过 HyperLink Navigator 快速获得一批活跃站点入口，省去逐个搜索和筛选的时间。

场景二：个人用户建立专属导航页面。普通用户可以将本项目 Fork 后，删除或注释掉不需要的链接，保留个人常用站点，并部署到 GitHub Pages 或 Vercel 作为私人浏览器起始页。

场景三：开发者学习外链数据结构的标准化建模。开发者可参考本项目的 JSON Schema，学习如何将非结构化的网址列表转换为带分类标签、时间戳和状态标记的结构化数据集。

场景四：企业内部用于整理可公开访问的测试站点列表。部分开发测试场景下需要借助外部站点模拟真实网络环境，本项目提供的链接列表可作为测试用例的来源参考。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境。请确保已安装 Git 和 Node.js 16 以上版本。

```bash
# 克隆仓库
git clone https://github.com/hyperlink-navigator/hyperlink-navigator.git
cd hyperlink-navigator

# 安装依赖（用于本地检测脚本）
npm install

# 运行本地预览服务，生成索引并启动静态服务器
npm run build
npm run serve
```

执行完成后，打开浏览器访问 http://localhost:3000 即可查看导航页面。若只需查看 Markdown 源文件，直接阅读 `docs/README.md` 或 `resources/index.md` 即可。

## 贡献指南

欢迎并感谢任何形式的贡献。请遵循以下步骤以确保项目的一致性和可维护性。

第一步：Fork 本仓库并在本地克隆您自己的副本，创建新分支进行修改。分支名称建议使用 `feature/xxx` 或 `fix/xxx` 格式。

第二步：若您需要新增资源链接，请在 `resources/index.md` 中按现有格式追加，并同步更新 `resources/index.json`。若您不熟悉 JSON 格式，可只修改 Markdown 文件，运行 `npm run generate-json` 自动生成。

第三步：所有新增链接必须通过可用性检测。提交前请运行 `npm run check-links` 确保新增 URL 返回状态码为 2xx 或 3xx。若链接暂时不可用，请添加 `[pending]` 标记并注明原因。

第四步：提交 Pull Request 之前，请确保所有单元测试通过。运行 `npm test` 执行全部测试套件，并确保文档中的链接与资源列表保持同步。

第五步：在 Pull Request 描述中清晰说明变更内容，包括新增链接的用途、分类标签和检测结果。若有移除或修改旧链接，请一并说明理由。

## 常见问题

问：本项目是否提供视频播放或下载功能？

答：不提供。HyperLink Navigator 仅收录外部站点的 URL 地址，本身不存储、不代理、不转发任何媒体内容。所有视频播放或下载行为发生在第三方源站，与本项目无关。

问：收录的链接失效了怎么办？

答：您可以自行在 `resources/index.md` 中删除或注释该链接，并提交 Pull Request。也可通过 Issues 页面报告失效链接，维护人员会在定期巡检中处理。项目每周自动运行一次可用性检测，异常链接会被标记但不会立即移除，以便观察是否临时波动。

问：我可以将本项目用于商业用途吗？

答：可以。本项目采用 MIT 许可证，您可以将代码和数据结构用于任何商业或非商业目的，无需征求作者许可。但请注意，项目收录的外部链接本身受各自站点的使用条款约束，请遵守第三方网站的规定。
