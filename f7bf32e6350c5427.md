# FilmLink Navigator

FilmLink Navigator 是一个面向影视从业者、内容创作者及技术研究人员的开源技术资源聚合与导航系统。本项目不提供任何流媒体播放、下载或存储服务，其核心定位为互联网公开技术文档、开发工具链、行业标准协议及学术研究论文的索引与结构化呈现。

项目目标用户包括电影特效工程师、实时渲染图形程序员、音视频编解码算法研究员以及分布式媒体处理系统的架构师。通过本导航系统，用户可以快速定位到特定领域的官方技术规范、开源代码仓库镜像、行业论坛的深度讨论贴以及公开的学术会议论文集，从而规避在信息海洋中无效检索的时间损耗。

本项目解决的核心痛点是影视技术领域信息碎片化严重、官方文档入口迁移频繁、中文技术社区缺乏权威外链沉淀的问题。我们通过人工筛选与社区提交相结合的方式，维护一份高质量、低死链的外链清单，并辅以标签系统与全文检索能力，确保技术资料的持久可溯性。

## 功能概览

- **智能分类索引**：根据影视制作全流程（前期、中期、后期、放映）对资源进行多级标签分类，支持按工序、按软件生态、按协议标准三种维度筛选。
- **站点健康度监测**：内置自动化巡检脚本，每日对收录的域名进行可达性及 SSL 证书有效性检测，并在界面中标记异常状态，降低用户访问无效资源的概率。
- **用户协同编辑**：支持 Pull Request 形式的资源增删改提议，经过核心维护者 Review 后合并，确保外链库随技术迭代持续进化。
- **暗色阅读模式**：针对长期在低光环境下工作的调色师与视效总监优化页面对比度与色温，减少视觉疲劳。
- **全文元数据检索**：不仅检索站点标题与描述，同时索引站点关联的技术栈关键词（如 OpenColorIO、USD、Alembic），实现基于语义的关联推荐。
- **RSS 订阅源生成**：针对每个分类生成独立的 RSS 订阅链接，方便用户通过阅读器聚合关注领域的最新收录动态。
- **导入/导出标准格式**：支持将导航数据导出为 JSON、YAML 或 OPML 格式，便于迁移至第三方阅读器或在团队内部共享。

## 应用场景

**场景一：独立特效师的技术选型调研**
独立特效师在筹备新项目的渲染管线时，可通过本导航的"渲染引擎"分类，快速对比不同开源渲染器的官方文档站与社区 Benchmark 数据。无需在搜索引擎中反复尝试关键词组合，即可获取经过社区验证的权威外链集合，将调研时间从数小时压缩至数分钟。

**场景二：高校影视技术实验室的教学资源索引**
高校教师在准备《数字图像处理》或《沉浸式媒体技术》课程大纲时，可将本导航作为学生查阅原版 Specification 的入口。通过导航中整理的 SMPTE 标准链接、ITU 建议书链接以及各大开源项目的 GitHub 镜像链接，学生可直接接触第一手技术资料，避免二手博客的错误传播。

**场景三：技术美术的 Shader 开发查表**
技术美术在编写 HLSL 或 GLSL 着色器时，经常需要查阅不同 GPU 厂商的指令集差异与精度规范。本导航的"图形 API"子分类聚合了 Vulkan、DirectX、Metal 及 OpenGL 的官方文档镜像站点，并附带各平台驱动更新的 Release Notes 链接，便于开发者在适配多平台时快速交叉验证。

**场景四：分布式媒体处理系统的架构评审**
后端工程师在设计云原生转码集群时，可通过本导航的"流媒体协议"分类查找 HLS、DASH、CMAF 等协议的规范原文及开源参考实现（如 FFmpeg、GPAC）。这些链接均指向项目官方仓库或标准化组织官网，确保架构设计依据的权威性与时效性。

## 快速开始

以下指令适用于 Linux/macOS 环境，Windows 用户建议通过 WSL2 或 Git Bash 执行。

```bash
# 步骤 1: 克隆项目仓库及子模块
git clone --depth 1 https://github.com/filmlink/navigator.git
cd navigator

# 步骤 2: 安装依赖（基于 Node.js 20 LTS 与 pnpm）
corepack enable
pnpm install --frozen-lockfile

# 步骤 3: 启动开发服务器（默认监听 3000 端口）
pnpm run dev
```

若需构建生产环境静态文件，请执行 `pnpm run build`，产物将输出至 `./dist` 目录，可使用任意静态服务器托管（如 `npx serve dist`）。

## 贡献指南

我们欢迎社区成员以多种形式参与本项目的建设，具体步骤如下：

1.  **报告问题或建议**：若发现某外链失效、分类错误或希望新增技术资源类别，请先在 Issues 中搜索是否已有相似话题。若无，请提交新 Issue，使用 `bug_report` 或 `feature_request` 模板，并附上相关链接及说明理由。
2.  **提交资源更新**：若您希望直接修改导航数据，请 Fork 本仓库，在 `./data/resources.json` 中按照 JSON Schema 规范添加或修改条目，并确保 `tags` 字段与现有分类体系一致。提交前请运行 `pnpm run lint:data` 验证数据格式。
3.  **发起 Pull Request**：完成修改后，从您的 Fork 向本仓库的 `main` 分支发起 PR。请确保 PR 标题简明扼要，描述中引用对应的 Issue 编号，并勾选 PR 模板中的自查清单（如“已通过本地构建测试”）。
4.  **参与巡检任务**：定期运行 `python scripts/health_check.py --full-scan` 并将生成的异常报告提交至 Issues，帮助维护者及时发现并清理死链。
5.  **完善技术文档**：若您发现 API 文档或部署指南存在歧义，欢迎直接修改 `./docs/` 目录下的 Markdown 文件并提交 PR。

## 常见问题

**问：导航站收录的资源是否经过人工审核？为何不直接使用自动化爬虫填充数据？**
答：是的，所有新提交的链接必须经过至少一名核心维护者的人工审核。我们主要评估站点的内容相关度（是否聚焦影视技术）、信息密度（是否包含有深度的技术文章或规范文档）以及站点的长期可维护性（是否有明确的更新日志或版本标注）。纯粹的自动化爬虫会产生大量低质量、高时效性的垃圾外链，增加用户的筛选成本，因此我们采用半自动化模式：爬虫仅用于辅助检测死链，收录决策仍由人工执行。

**问：如何请求删除某个已收录的链接？**
答：若您发现已收录的链接存在恶意软件、钓鱼内容或频繁弹窗广告等严重侵害用户权益的行为，请立即通过 Issue 报告，并选择 `security` 标签。我们承诺在收到报告后的 24 小时内进行核实，若确认违规，将第一时间从索引库中移除该条目并同步更新 RSS 订阅源。对于因内容观点不同产生的争议链接，我们建议您通过社区论坛发起讨论，而不直接请求删除。

**问：本地开发时，健康度检测脚本提示 SSL 证书验证失败，如何解决？**
答：这通常是因为本地环境缺少目标站点的中间证书链，或者系统时间不准确所致。首先请使用 `ntpdate` 或系统时间同步功能校准本地时钟。若问题仍然存在，可在 Python 脚本中设置 `verify=False` 参数（仅限调试环境），或通过 `--ignore-ssl` 标志运行检测脚本。需要强调的是，生产环境的巡检服务必须启用严格的证书验证，以确保用户访问的安全性与合规性。
