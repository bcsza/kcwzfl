# LinkHub

LinkHub 是一个面向开发人员、技术研究人员及数字内容创作者的轻量级外链资源导航与聚合工具。项目定位为“技术资源的外链汇总站”，旨在解决个人收藏夹混乱、团队共享链接困难、以及技术调研过程中信息碎片化的问题。LinkHub 不存储任何实体资源，仅提供结构化的外部链接索引，帮助用户快速定位到高质量的技术文档、设计素材、影音参考及行业动态站点。

本项目适用于需要维护公共书签库的团队、希望系统化管理个人学习路径的开发者，以及需要对外输出精选资源列表的内容创作者。LinkHub 基于纯静态 Markdown 与 JSON 数据驱动，可无缝集成至现有的 CI/CD 工作流，支持一键导出为 HTML 门户或 API 接口，便于嵌入内部知识库或第三方服务平台。

## 功能概览

- 分类导航体系：支持多级目录与标签系统，用户可按技术领域、资源类型或使用场景对链接进行精细化分类。
- 一键快速添加：通过标准化 JSON 模板即可新增资源条目，无需修改核心代码，降低维护成本。
- 全文检索能力：集成客户端全文搜索，支持对链接标题、描述、标签及分类进行即时过滤。
- 数据导入与导出：提供 CSV 与 JSON 格式的批量导入导出接口，方便从现有书签系统迁移数据。
- 定期链接可用性检查：内置链接状态检测脚本，可定期输出失效链接报告，保证资源质量。
- 响应式展示面板：自动适配桌面端与移动端，支持列表视图与卡片视图切换，提升浏览体验。
- 访问统计与点击追踪：集成轻量级事件追踪接口，支持统计各资源链接的点击热度（可选开关）。
- 夜间模式与主题定制：内置明暗两套主题，并开放 CSS 变量接口，方便二次定制品牌风格。

## 应用场景

个人技术知识库构建：开发者可将日常阅读的博客、官方文档、在线工具等链接统一收录至 LinkHub，并按照编程语言、框架或研究方向进行归类，形成长期积累的个人技术知识外链库，替代浏览器自带的混乱书签栏。

团队内部资源协同：研发团队或设计团队可利用 LinkHub 搭建内部共享导航页，集中存放项目相关的设计规范、组件库文档、测试环境地址及运维手册，新成员入职时可快速了解团队常用工具链。

技术社区内容聚合：开源社区或技术论坛的管理员可使用 LinkHub 聚合社区内推荐的优质外部资源，例如视频教程、电子书、业界新闻源等，作为社区官网的扩展模块，丰富内容生态。

影音与创意参考收集：内容创作者或多媒体研究者可借助 LinkHub 分类管理不同风格的影视素材站、音乐音效库、视觉灵感图库，方便在项目创作周期内快速检索参考素材。

## 快速开始

以下步骤将帮助您在本地环境中快速启动 LinkHub 服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkhub-io/linkhub.git

# 进入项目根目录
cd linkhub

# 安装项目依赖（使用 npm）
npm install

# 启动开发服务器，默认监听端口 3000
npm run dev
```

执行上述命令后，在浏览器中访问 `http://localhost:3000` 即可查看 LinkHub 主界面。若需构建生产环境静态文件，请使用 `npm run build` 命令，输出目录为 `dist/`。

## 贡献指南

我们欢迎并感谢任何形式的贡献，无论是提交 Issue、改进文档还是贡献代码。请遵循以下步骤参与本项目。

第一，在 GitHub 仓库页面点击 `Fork` 按钮，将本项目复制至您的个人账户下，并克隆该复刻版本至本地开发环境。

第二，创建新的功能分支，分支名称应清晰反映您要修复的问题或新增的功能，例如 `fix/search-filter-bug` 或 `feature/add-dark-mode`。

第三，完成代码或文档修改后，运行本地测试套件 `npm run test` 确保所有现有功能未受影响，并针对新增功能补充对应的单元测试或 e2e 测试用例。

第四，提交代码时请遵循约定式提交规范，使用 `feat:`、`fix:`、`docs:`、`chore:` 等前缀编写清晰的提交信息，便于自动化生成变更日志。

第五，向主仓库的 `main` 分支发起 Pull Request，并在 PR 描述中详细说明变更内容、关联的 Issue 编号以及测试覆盖情况。PR 合并前需要通过所有 CI 检查项及至少一名维护者的代码评审。

## 常见问题

问：LinkHub 是否存储用户上传的文件或视频内容？

答：不存储。LinkHub 仅保存外部链接的标题、描述、分类标签及 URL 字符串本身。所有资源均指向第三方站点，项目本身不托管任何实体文件或媒体流。用户点击链接后将跳转至原始外部站点，相关内容的版权与可用性由源站点负责。

问：如何批量导入我现有的浏览器书签或收藏夹？

答：LinkHub 支持通过 JSON 或 CSV 格式批量导入数据。您可以从 Chrome、Firefox 等浏览器导出书签为 HTML 文件，再使用项目提供的转换脚本 `scripts/convert_bookmarks.py` 将 HTML 转换为 LinkHub 识别的 JSON 格式，最后将数据复制至 `src/data/sources.json` 文件中。详细转换步骤请参考文档 `/docs/user-guide/import-export.md`。

问：链接检测脚本提示大量失效链接，我应该如何处理？

答：链接检测脚本默认以 HEAD 请求方式检查每个 URL 的 HTTP 状态码。对于返回 4xx 或 5xx 状态的链接，脚本会在终端输出表格报告。您可以根据报告手动在 `sources.json` 中删除或更新对应条目。若某站点临时维护导致误报，可配置脚本忽略特定域名，具体方法见脚本头部注释。
